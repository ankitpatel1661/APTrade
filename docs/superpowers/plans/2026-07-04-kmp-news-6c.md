# Plan: Increment 6c — News (shared core + desktop tab)

Spec: docs/superpowers/specs/2026-07-04-kmp-news-6c-design.md (binding parity facts).
Branch: kmp-news-6c off main @ ac2d5c2. Baselines: shared 122 / android 13 /
desktop 107 / Swift 193 / xcframework 3 slices.

## Global Constraints (binding for every task)

- Clean Architecture: commonMain framework-free apart from the sanctioned Ktor layer;
  no desktop/AWT imports in :shared; PDF/AWT stay in desktop infra.
- CancellationException rethrown FIRST in every catch, everywhere. Use-case error
  swallow (≡ macOS `try?`) must still rethrow CancellationException.
- ViewModels Main-confined (plain vars legal; document like existing VMs).
- Esc ownership chain untouched (palette → panel/dialogs self-consume).
- Money rules unaffected (no money in this increment) — do not touch money code.
- The Finnhub API key: query param `token`; NEVER logged, NEVER committed, NEVER
  echoed into reports (report "key present/absent" only).
- Standing waiver: UI composition ships without unit tests; pure logic is tested.
- Suites run with --rerun-tasks and XML-counted. Swift/iOS: DEVELOPER_DIR prefix;
  scheme APTradeLite-Package ARCHS=arm64; park a shadowing .xcodeproj if present.
- JDK: JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home;
  ./gradlew only.
- HARD RULE for implementers: no Agent-tool delegation; the dispatched agent does the
  work itself.

### Task 1: Shared news domain — types, ports, DTO + mapper

Files (new, commonMain): domain/NewsCategory.kt, domain/NewsArticle.kt,
domain/NewsPorts.kt (NewsRepository + BookmarkStore interfaces),
infrastructure/FinnhubNewsDTO.kt (DTO + mapper). Tests: commonTest FinnhubNewsMapperTest.

1. `enum class NewsCategory(val finnhubValue: String, val displayName: String)`:
   General("general","General"), Crypto("crypto","Crypto"), Merger("merger","Merger") —
   declaration order is the UI order.
2. `data class NewsArticle(id: String, headline: String, summary: String, source: String,
   url: String, imageUrl: String?, publishedAtEpochSeconds: Long, category: String?,
   relatedSymbol: String?)` — plain value type, invariants live in the mapper
   (spec divergence: strings not URL/Date types).
3. Ports: `interface NewsRepository { suspend fun marketNews(category: NewsCategory):
   List<NewsArticle>; suspend fun companyNews(symbol: String): List<NewsArticle> }`;
   `interface BookmarkStore { suspend fun load(): List<NewsArticle>; suspend fun
   save(articles: List<NewsArticle>) }`.
4. DTO `FinnhubArticleDTO` — ALL fields nullable with defaults: category, datetime
   (Double — Finnhub sends numeric seconds; convert via toLong()), headline, id (Long?),
   image, related, source, summary, url. kotlinx-serialization, ignoreUnknownKeys via
   a file-local Json instance (match YahooDTO file conventions).
5. Mapper `FinnhubNewsMapper.article(dto): NewsArticle?` — null (drop) when headline
   blank/absent OR url blank/absent; summary/source default ""; image/related "" → null;
   id = id?.toString() ?: url; publishedAt = datetime?.toLong() ?: 0L; category/related
   passed through. `articles(list) = list.mapNotNull(::article)`.
6. Tests (≥8): happy-path full mapping; drop-on-blank-headline; drop-on-missing-url;
   ""-image → null; id fallback to url; datetime absent → 0L; summary/source default "";
   list mapNotNull keeps order of survivors.

### Task 2: Shared Finnhub repository + use cases

Files (new, commonMain): infrastructure/FinnhubNewsRepository.kt,
application/NewsUseCases.kt (FetchMarketNews, FetchCompanyNews, LoadBookmarks,
ToggleBookmark). Tests: commonTest with Ktor MockEngine (mirror YahooMarketDataRepository
test conventions) + pure toggle tests.

1. `FinnhubNewsRepository(apiKey: String, client: HttpClient = defaultYahooHttpClient()…)`
   — reuse the existing per-platform engine helper (rename-neutral: whatever
   YahooMarketDataRepository uses for its default client; do NOT create a second
   expect/actual). Base https://finnhub.io/api/v1.
   - marketNews: GET /news, params category=finnhubValue, token=apiKey.
   - companyNews: GET /company-news, params symbol=symbol.uppercase(), from/to =
     UTC yyyy-MM-dd, to=now, from=now−7*24*3600 (kotlinx-datetime or epoch math —
     match whatever :shared already uses for dates; NO java.time in commonMain).
   - Error mapping ≡ the Yahoo adapter's shape: 429 → QuoteError.RateLimited, other
     non-2xx/malformed → QuoteError.Network, CancellationException rethrown. (Reuse
     QuoteError — the established shared error taxonomy; do NOT invent a NewsError.)
2. Use cases:
   - `FetchMarketNews(repository)` / `FetchCompanyNews(repository)`: execute returns
     List<NewsArticle>; SWALLOW all failures to emptyList() (≡ macOS try?) but rethrow
     CancellationException FIRST. @Throws NOT needed (they don't throw).
   - `LoadBookmarks(store)`: store.load().
   - `ToggleBookmark(store)`: `execute(article, current: List<NewsArticle>):
     List<NewsArticle>` — if any element has the same id → filter it out; else new list
     with article at index 0; then store.save(result); returns result. Pure logic +
     persistence in one use case (macOS shape).
3. Tests (≥7): marketNews URL/params assembled exactly (MockEngine capture, token
   present); companyNews symbol uppercased + from/to format + 7-day window (fixed clock
   seam — inject a clock/now provider, default real); 429 → RateLimited; use-case
   swallow → emptyList on Network error; use-case rethrows CancellationException
   (cancelling fake); toggle removes-by-id; toggle inserts-at-0 and saves.
4. `compileKotlinMacosArm64` green (no xcframework assembly — Task 7 does that).

### Task 4: Desktop designkit — relative time + async thumbnail loader

Files (new): designkit/RelativeTime.kt, designkit/RemoteImage.kt (loader + composable).
Tests: RelativeTimeTest (pure, exhaustive); loader logic test if a pure seam exists.

1. `relativeTimeText(epochSeconds: Long, nowEpochSeconds: Long): String` (pure, both
   params explicit): <60s "just now"; <60m "N minutes ago" (1 minute ago singular);
   <24h "N hours ago" (singular at 1); <7d "N days ago" (singular at 1); else absolute
   "MMM d" (Locale.US, UTC). Document the macOS named-relative divergence.
2. RemoteImage: `@Composable fun RemoteThumbnail(url: String?, modifier: Modifier)` —
   null url → fallback box (DK.surfaceHi, 8dp rounded); else launch (keyed on url) a
   fetch via a small shared HttpClient (desktop CIO/JVM — reuse or construct ONE
   process-wide client in the file, closed never — document), decode
   org.jetbrains.skia.Image.makeFromEncoded → toComposeImageBitmap, LRU cache (~64,
   LinkedHashMap accessOrder synchronized) keyed by url; ANY failure → fallback box.
   CancellationException-first in the fetch catch.
3. Tests: relativeTimeText ≥10 pinned cases incl. boundaries (59s/60s, 59m/60m,
   23h/24h, 6d/7d, singulars, absolute fallback format). LRU eviction pinned via the
   cache object if it's exposed internal (make the cache `internal` for testability).
